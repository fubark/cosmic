Cosmic

Cosmic is a simple Javascript and WASM runtime for developing Desktop, Gaming, Backend, and Web applications. Our approach is to start with V8 (the fastest script engine) and design a rich API and tools around it. To get a gauge of what Cosmic can do now, check out the docs for each API module as well as the Tools section.

Native API

One of the goals of Cosmic is to provide a cross platform Js/Wasm API to do all the neat things your computer can do. This means having a window API, a 2D/3D graphics API, a multimedia API, a networking API, a file system API, and more. On top of these basic layers, Cosmic will also provide higher level constructs like a UI framework, an animation API, and more.

Supporting the Web/Node API is a non goal, but we do want to enable similar capabilities. For example, we have implemented "fetch" as making http requests from the cs.http module. Once the Cosmic API is fleshed out more, it shouldn't be hard for one to create a Web polyfill in pure JS.

Conventions

If you've come from Web or Node development you might be wondering how different it is to code in Cosmic:

  • Importing: We want to support ES modules as the way to import code from local and the web. This is currently not implemented but it is a priority. Support for node npm packages is a non goal and will only be considered depending on demand.
  • Async: All API functions that have an async alternative are synchronous by default. For example, cs.http.get() is synchronous and the async version will be cs.http.getAsync(). Code having clear intentions is a goal for Cosmic. One should feel that they can accomplish anything synchronously and only reach for async when they are ready to or planned to.
  • Errors: Synchronous functions will never throw an exception. null will be returned instead and a follow up errCode() or errString() will get you info about the error. Exceptions are evil and is an example of hidden control flow we wish to avoid in Cosmic. Async promises will be rejected with the error info. errCode and errString are not currently implemented but they are priority features.

We are aiming to make coding accessible, unrestrictive, productive, fast, and fun! Like what you see? Join our community and voice your opinions!

Getting Started

Hello World

Open up your text editor and create hello.js:

const Color = cs.graphics.Color

printLine('Hello World!')

const w = cs.window.create('Demo', 1200, 720)
w.onUpdate(g => {
    g.setFontSize(52)
    g.setFillColor(Color.blue)
    g.text(400, 300, 'Hello World!')
})

Then run this in the terminal:

cosmic run hello.js
